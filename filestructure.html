Basic File Structure

When in doubt, write your code in Robot.java

For beginning purposes, code should be written in the /src/.../ Robot.java file in the appropriate method.

{Write in the Curly Brackets}

Code is (almost) always written within the curly brackets of a method.

Import Space

The section highligted in pink is where you can import all of the necesarry libraries. For example, the line import edu.wpi.first.wpilibj.TimedRobot;

The Shelf

Although not an official term, the shelf is the space within the curly brackets of a class, but outside of a method. The shelf is used for defining objects that must be accessed in multiple methods. This allows them to be visible everywhere in the code (effectively). You cannot call methods from the shelf.

Init vs. Periodic

An init method is called at the beginning of a certain time period and only runs once. This is a good place to initialize necesarry variables or reset entries that only need to happen once. At the beginning of the teleop periodic, the init method is only called once. The periodic methods are called over and over. For example, the teleop periodic runs repeatedly whenever the robot is in teleop mode.

Robot Methods

These methods are called whenver the robot is enabled. This is a good time to service things such as pneumatics or read values that must be monitored always.

Autonomous Methods

These methods only run for the autonomous period.

Teleop Methods

These methods only run for the autonomous period.

Disabled Methods

We don't often use these methods.

Test Methods

This can be used to test certain functionality without it having to running during a match.
