Working with Motors

Importing the Libs

Add the following line in your import space: import com.revrobotics.*;

How do I create a motor object?

The CANSparkMax object is found under the CANSparkMax class. It takes two parameters: (canID, MotorType). The first parameter is the canID (an int), set within motor controller. The second parameter is an enum . The two types, MotorType.kBrushed and MotorType.kBrushless, are used to select the motor type. If we're working with a NEO, use the brushless type. Otherwise, use brushed! In full, the code looks like this:

CANSparkMax m1 = CANSparkMax(1, MotorType.kBrushless);

Moving the Motor

To bring the motor to a set point, you can utilize the method, .set(d), where d is a value from -1 to 1. For example, the code m1.set(.3) sets the m1 motor object to 30%.

What if the motor is going the wrong direction

Add this line: m1.setInverted(true);

Drive Train Controls

Imports

Add this line to the import space: import frc.robot.drive.DriveTrain;

Drive Train Object Creation

A drive train object is created through the following line. DriveTrain dt = new DriveTrain(i1,i2,i3,i4); It takes four parameters which are the ids of each drive motor. i1 and i2 are the ids of the front and rear left drive motors and i3 and i4 are the ids front and rear right drive motors.

Joystick Controls

Creating an XBox Controller

First, place this statement into your imports: import edu.wpi.first.wpilibj.XboxController; The controller object takes one int as a parameter. This is the specific port on the driver station. To declare a controller at index 0, use the following code:

XboxController xb = XboxController(0);

Methods for Getting Buttons

A complete reference of all of the methods can be found here.

An important note: There are 3 different ways to read a button. The first, .get[ specific button], reads the value of a button at the instant of reading. The other two go hand in hand, the get pressed and get released. This is helpful when trying to initiate a task or stop it. Rather than reading the value every time, the getPressed and getReleased methods allow for safer starting and stopping of tasks that need to start and end in precise ways.

Referencing Joystick Position

Once again, a complete reference of all of the methods can be found here.

The two methods that we will use most often are xboxObject.getLeftY(); and xboxObject.getRightY(); These methods return a double from -1 to 1 signifying the position of the respective joystick on the XBoxController.
